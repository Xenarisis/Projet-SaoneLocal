# Weekly Reports

Chaque weekly est rédigé en Markdown et suit une structure en cinq parties : un résumé rapide de la semaine, le détail des tâches réalisées et des livrables produits, les retours reçus sur ces livrables, les blocages rencontrés avec les solutions envisagées, et enfin les tâches prévues pour la semaine suivante. Les fichiers sont nommés selon le format `YYYY-WXX.md` (ex : `2025-W17.md`) pour rester ordonnés chronologiquement.

---

## Structure des fichiers

```
weekly/
├── README.md       ← ce fichier
├── YYYY-WXX.md     ← template vide
├── 2025-W17.md
├── 2025-W18.md
└── ...
```

## Template

Chaque fichier weekly suit ce modèle :

```markdown
# Weekly — Semaine XX · YYYY

---

## Résumé rapide

> Une phrase pour décrire l'avancement global de la semaine.

## Ce qui a été fait

### Tâches réalisées
- [ ] Tâche 1
- [ ] Tâche 2

### Livrables produits
| Livrable | Type | Statut | Commentaires reçus |
|---|---|---|---|
| Maquette accueil | Figma | ✅ Validé | ... |
| Composant navbar | Code | 🔄 En cours | ... |

## Retours & évaluations

### Retours reçus (prof / client / équipe)
- **Sur quoi** : Maquette page d'accueil
  - Retour : _"..."_
  - Action à faire : ...

## Blocages & problèmes

| Problème | Contexte | Solution envisagée | Résolu ? |
|---|---|---|---|
| ... | ... | ... | ❌ |

## Prévu pour la semaine prochaine

- [ ] Tâche prioritaire 1
- [ ] Tâche prioritaire 2

---
_Rédigé le : JJ/MM/AAAA_
```
